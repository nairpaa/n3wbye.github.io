Introduction to Frida

Dasar frida untuk pentest Android.

Frida adalah alat intrumentasi yang dinamis dan fleksibel untuk developer, reverse-engineers dan security researchers. Dengan tool ini kita bisa memodifikasi kode flow eksekusi atau hanya sebatas mengamati keadaan proses dalam aplikasi.

Frida sangat berguna karena ia dapat menyuntikkan “sesuatu” ke dalam proses yang sedang berjalan di berbagai platform: Android, iOS, Windows, Mac dan lain-lain.

Pada tutorial kali ini saya akan menjelaskan dasar penggunaan frida untuk pentest aplikasi Android.

APK yang digunakan adalah AndroidLab buatan Ironhackers. Pada aplikasi tersebut terdapat 4 soal yang harus diselesaikan menggunakan frida.

1. Requirements#

  1. Device atau emulator Android
    • Jika Anda tidak memiliki akses root, Anda bisa mengikuti tutorial di sini.
    • Salah satu emulator Android yang saya rekomendasikan adalah Genymotion.
  2. Android Debug Bridge
    • ADB adalah command line tool serbaguna yang memungkinkan kita berkomunikasi dengan perangkat Android.
    • ❯ apt install android-tools-adb
    • Install APK yang akan digunakan sebagai bahan praktik.
    • ❯ adb install AndroidLab.apk
  3. Frida tools
    • Pastikan bahwa frida terinstall pada komputer Anda.
    • ❯ pip3 install frida-tools
    • ❯ frida --version
  4. Frida server pada Android
    • Sebelum mendownload frida-server, pastikan bahwa versinya sama dengan yang terinstall pada komputer Anda dan pastikan juga sesuai dengan arsitektur Android yang digunakan.
    • ❯ adb shell getprop ro.product.cpu.abi
    • Selanjutnya, ekstrak dan upload frida-server ke Android.
    • ❯ adb devices -l
❯ adb push frida-server /data/local/tmp/
❯ adb shell
android:/ $ chmod +x /data/local/tmp/frida-server
  5. Reverse APK tools
    • Reverse APK tools digunakan untuk melihat source code pada APK yang akan diuji.
    • Untuk lebih jelasnya Anda bisa membaca artikel saya tentang ini.

Setelah semua kebutuhan terpenuhi, kita akan langsung mencoba menggunakan frida.

2. Exercise 1 - Call me!#

Untuk mengakses soal yang pertama ini, pada aplikasi pilih menu Level 1 - Call me!.

Jika kita amati source code yang sudah di-reverse sebelumnya, terdapat class Callme yang kurang lebih potongan kodenya seperti berikut:

public void onCreate(Bundle savedInstanceState) { // fungsi ini akan dipanggil ketika activity dijalankan
  super.onCreate(savedInstanceState); // menjalankan kode selain yang ada di onCreate() dari class induk
  setContentView(R.layout.activity_callme); // menampilkan layout activity_callme
  setSupportActionBar((Toolbar) findViewById(R.id.toolbar)); // toolbar
  this.msg = (TextView) findViewById(R.id.flag); // menampilkan text dengan id flag ("Nothing to Show")
}

private void call_me_win() { // fungsi yang tidak pernah dipanggil
  this.msg.setText(R.string.flag1); // merubah nilai pesan yang ditampilkan dengan variable string flag1 ("c4ll_m3_fl4g")
}

Untuk menyelesaikan soal ini, kita perlu memanggil fungsi call_me_win() dengan menggunakan frida.

Pertama-tama, jalankan frida-server pada Android.

❯ adb shell
android:/ $ su
android:/ # ./data/local/tmp/frida-server

Selanjutnya, pada komputer jalankan perintah berikut untuk mengetahui identifier dari aplikasi AndroidLab:

❯ frida-ps -Ua | grep "AndroidLab"
30246  AndroidLab     com.ironhackers.androidlab

Setelah mengetahui identifier dan flow proses aplikasi, kita akan buat script untuk memanggil fungsi call_me_win().

Simpan script berikut ke dalam file callme.js:

Java.perform(function () { // memastikan thread saat ini terpasang dan memanggil fungsi Java.choose()
  Java.choose('com.ironhackers.androidlab.Callme', { // enumerasi live instance dari class com.ironhackers.androidlab.Callme dengan men-scan heap Java dan callback berisi objek yang ditentukan
    onMatch: function(instance) { // fungsi ini mengembalikan stop string untuk membatalkan enumerasi diawal
      Java.scheduleOnMainThread(function () {  // menjalankan fungsi call_me_win() pada thread utama
        instance.call_me_win();
      });
    },
    onComplete: function() {} // dipanggil ketika semua kelas telah dienumerasi
  });
});

Jalankan perintah berikut pada komputer Anda untuk menyisipkan script yang telah dibuat ke dalam proses yang sedang berjalan:

❯ frida -U --no-pause -l callme.js com.ironhackers.androidlab

  • -U: menghubungkan dengan perangkat USB.

Terlihat di atas bahwa fungsi call_me_win() berhasil dijalankan dan flag-nya tampil di layar.

3. Exercise 2 - AlwaysTrue#

Untuk mengakses soal yang kedua, pada aplikasi pilih menu Level 2 - AlwaysTrue.

Jika kita amati source code yang sudah di-reverse sebelumnya, terdapat class Alwaystrue yang kurang lebih potongan kodenya seperti berikut:

public void onCreate(Bundle savedInstanceState) { // fungsi ini akan dipanggil ketika activity dijalankan
  super.onCreate(savedInstanceState); // menjalankan kode selain yang ada di onCreate() dari class induk
  setContentView(R.layout.activity_alwaystrue); // menampilkan layout activity_alwaystrue
  setSupportActionBar((Toolbar) findViewById(R.id.toolbar)); // toolbar
  if (imposible_check()) { // apabila fungsi imposible_check() mengembalikan nilai true, flag2 ("4lw4ystru3_fl4g") akan ditampilkan
    ((TextView) findViewById(R.id.msg)).setText(R.string.flag2);
  }
}

private boolean imposible_check() {
  return false;
}

Untuk menyelesaikan soal ini kita perlu membuat fungsi imposible_check() mengembalikan nilai true.

Buat lah file alwaystrue.js dan isikan dengan script berikut:

Java.perform(function(){ // memastikan thread saat ini terpasang dan memanggil fungsi Java.use() pada variable alwastrueactivity
  var alwaystrueactivity= Java.use('com.ironhackers.androidlab.Alwaystrue'); // Javascript wrapping pada class com.ironhackers.androidlab.Alwaystrue
  alwaystrueactivity.imposible_check.implementation = function () { // membuat fungsi imposible_check() mengembalikan nilai true
    return true;
  }
});

Jalankan perintah berikut pada komputer Anda untuk menyisipkan script yang telah dibuat ke dalam proses yang sedang berjalan:

Buat lah file createme.js dan isikan dengan script berikut:

Java.perform(function(){

  Java.choose('com.ironhackers.androidlab.Createme', { // enumerasi live instance dari kelas com.ironhackers.androidlab.Callme dengan men-scan heap Java dan callback berisi objek yang ditentukan

      instance.person.value.setWallet(wallet.$new(999)); // mengatur nilai wallet menjadi 999
    },
    onComplete: function() {} // dipanggil ketika semua kelas telah dienumerasi
  });
});

Buat lah file sniff.js dan isikan dengan script berikut:
