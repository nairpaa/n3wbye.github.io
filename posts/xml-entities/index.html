XML Entities

Mengenal entitas XML.

Pada kesempatan kali ini saya akan mencoba menjelaskan beberapa fitur utama XML yang relevan untuk memahami kerentanan XXE.

1. Apa itu XML?#

XML adalah singkatan dari extensible markup language. XML adalah bahasa yang dirancang untuk menyimpan dan mengirimkan data. Seperti HTML, XML menggunakan tree-like structure tag dan data.

Tidak seperti HTML, XML tidak menggunakan tag yang telah ditentukan sebelumnya, sehingga tag dapat diberi nama yang menggambarkan data. Sebelumnya dalam sejarah web, XML sedang populer sebagai format transportasi data (“X” dalam “AJAX” adalah singakatan dari “XML”). Namun popularitasnya kini menurun karena format JSON.

2. Apa itu entitas XML?#

Entitas XML adalah cara untuk merepresentasikan item data dalam dokumen XML, alih-alih menggunakan data itu sendiri.

Berbagai entitas dibangun dengan spesifikasi bahasa XML. Misalnya, entitas &lt; dan &gt; mewakili karakter < dan >. Ini adalah metakarakter yang digunakan untuk menunjukkan tag XML, dan karenanya secara umum harus direpresentasikan menggunakan entitasnya saat muncul di dalam data.

3. Apa itu document type definition (DTD)?#

Document type definition XML (DTD) berisi deklarasi yang dapat menentukan struktur dokumen XML, tipe nilai data yang dapat dikandungnya, dan item lainnya. DTD dideklarasikan dalam elemen DOCTYPE opsional di awal dokumen XML.

DTD dapat sepenuhnya mandiri di dalam dokumen itu sendiri (dikenal sebagai “internal DTD") atau dapat dimuat dari tempat lain (dikenal sebagai “external DTD") atau dapat merupakan gabungan dari keduanya.

4. Apa itu XML custom entities?#

XML memungkinkan entitas kustom untuk didefinisikan dalam DTD. Sebagai contoh:

<!DOCTYPE foo [ <!ENTITY myentity "my entity value" > ]>

Definisi ini berarti bahwa setiap penggunaan referensi entitas &myentity; dalam dokumen XML akan diganti dengan nilai yang ditentukan: “my entity value”.

5. Apa itu XML external entities?#

XML external entities adalah tipe entitas kustom yang definisinya terletak di luar DTD tempat mereka dideklarasikan.

Deklarasi entitas eksternal menggunakan kata kunci SYSTEM dan harus menentukan URL dari mana nilai entitas harus dimuat. Sebagai contoh:

<!DOCTYPE foo [ <!ENTITY ext SYSTEM "http://normal-website.com" > ]>

URL dapat menggunakan protokol file://, sehingga entitas eksternal dapat dimuat dari file. Sebagai contoh:

<!DOCTYPE foo [ <!ENTITY ext SYSTEM "file:///path/to/file" > ]>

XML external entities menyediakan sarana utama di mana serangan XML external entity muncul.
