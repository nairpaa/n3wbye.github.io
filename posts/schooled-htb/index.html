3.1. Check the moodle version#

Ada beberapa cara yang saya lakukan untuk menemukan versi Moodle yang digunakan, yaitu:

  1. Melihat source code Moodle di github resmi dan menemukan file /theme/upgrade.txt.
  2. Tools scanner Moodle (moodlescan.py) melakukan pengecekan versi dengan beberapa cara, salah satunya mengakses /question/upgrade.txt.

Dengan kedua cara tersebut saya menemukan versi Moodle yang digunakan adalah versi 3.9.

3.2. Look for related vulnerabilities#

Setelah mengetahui versi Moodle, saya mencari tahu lebih dalam tentang kerentanan yang ada pada versi tersebut di web resmi Moodle.

Berikut adalah beberapa kerentanan yang tersedia di Moodle versi 3.9:

  • CVE-2020-14320: MSA-20-0008: Reflected XSS in admin task logs filter
  • CVE-2020-14321: MSA-20-0009: Course enrolments allowed privilege escalation from teacher role into manager role
  • CVE-2020-25627: MSA-20-0011: Stored XSS via moodlenetprofile parameter in user profile
  • CVE-2020-25628: MSA-20-0012: Reflected XSS in tag manager
  • CVE-2020-25629: MSA-20-0013: “Log in as” capability in a course context may lead to some privilege escalation
  • CVE-2020-25698: MSA-20-0016: Teacher is able to unenrol users without permission using course restore
  • CVE-2020-25699: MSA-20-0017: Privilege escalation within a course when restoring role overrides
  • Dan lain-lain

Yang menarik adalah CVE-2020-14321 yang diklaim bisa RCE. Untuk mengeskploitasi kerentanan ini saya memerlukan akun teacher.

3.3. Register a moodle account#

Karena semua kerentanan memerlukan akun, saya mencoba membuat akun student baru.

3.4. Enroll in math course#

Pada kursus matematika terdapat announcements dari akun teacher Manuel Philips yang mengharuskan kita men-set MoodleNet. Ia akan melakukan pengecekan MoodleNet setiap siswanya.

Pada informasi yang didapatkan sebelumnya terdapat kerentanan XSS pada MoodleNet (CVE-2020-25627).

Tidak ada mekanisme pengamanan cookie membuat saya sebagai penyerang dapat memanfaatkan kerentanan ini untuk mengambil cookie milik teacher dan mengeksploitasi CVE-2020-14321 untuk mendapatkan RCE.

5.3. Privilege escalation#

Selanjutnya saya mencari tahu ID dari akun teacher yang digunakan dengan mengakses menu profil.

ID=24

Pada request yang disimpan sebelumnya, saya melakukan perubahan pada parameter userlist[] menjadi ID dari teacher yang digunakan (24) dan roletoassing menjadi 1. Hal ini dilakukan untuk menambahkan user teacher menjadi bagian dari kursus dengan role manager.

Hasilnya:

Klik akun Lianner Carter dan klik “Log in As Administrator”.

Terlihat pada gambar di bawah saya berhasil melakukan privilege escalation ke role manager, tetapi tidak bisa melakukan instalasi plugin untuk mengeksploitasi RCE.

7.2. Via PKG repository manipulation#
